## OSPF Bridge Simulation Engine

#### 项目简介

本项目是一个基于 Python 和 Open vSwitch (OVS) 的网络仿真引擎，支持在 Ubuntu 虚拟机中构建虚拟路由器集群，并实现了纯虚拟 OSPF 网络与物理宿主机 (Windows) 的全网双向连通。

#### 环境依赖

操作系统：Ubuntu 虚拟机 (建议配置双网卡，实现管理与业务通道的物理隔离)。

核心组件：Python 3.x 运行环境，Open vSwitch (OVS) 虚拟交换机。

宿主机环境：Windows 操作系统，需配合系统自带的 route 命令添加静态路由，以打通跨界网段。

#### 使用说明

1.规划拓扑：打开项目根目录下的 topology.yaml 文件，根据实验需求定义路由器节点、网段划分与网线连接关系。

2.启动网络：在 Ubuntu 终端中进入项目根目录，使用管理员权限执行主程序，一键拉起整个虚拟网络拓扑：
sudo python3 main.py

3.登录设备：拓扑成功拉起后，引擎会自动为每个虚拟路由器和 PC 分配本地端口 (例如 5001, 5002)。在宿主机或虚拟机内通过 telnet localhost 端口号 即可登录对应设备的命令行控制台。

4.销毁环境：实验测试完成后，执行配套的清理指令或停止脚本，系统将自动拆除底层的 OVS 网桥和 veth 虚拟网线，确保系统环境零残留。
